# Pantheon Newsroom: Engineering Log (2026-03-27 18:58)
## Objective: Pipeline Stability, API Control, and State Accuracy

This document summarizes the changes made to the Pantheon Newsroom codebase to resolve API flooding, dashboard state inaccuracies, and article rejection issues.

---

### 1. Unified Pipeline Control & Safety
- **Master Guardrail**: Implemented a "Standby Check" in `lib/pipeline.ts` that immediately blocks all background/scheduled runs if `isLive` is set to `false`. This prevents unauthorized token consumption by the scheduler.
- **Manual Override**: Updated the pipeline trigger API (`/api/process-news`) and the core `runPipelineCycle` to support a `isManual` flag. This allows manually clicking "RUN NOW" to bypass the standby guardrail for testing, while the automatic scheduler remains blocked.
- **Kill Process Sync**: Refined the `AbortController` integration to ensure that both the UI and the network requests (to Anthropic) terminate instantaneously when a user aborts.

### 2. Dashboard State Representation
- **Real-time Status Polling**: Integrated the actual background process status (`pipelineRunning`) into the standard `/api/settings` polling cycle (every 10 seconds).
- **Decoupled Log Inference**: Removed the fragile 8-second auto-reset logic in `OperationMap.tsx`. The dashboard now relies on the **actual backend state** for its "ACTIVE" vs "STANDBY" visual indicators, rather than guessing based on log activity.
- **Robust UI Recovery**: Ensured the "ABORTING..." and "TRIGGERING" transient states in `MasterControls.tsx` are correctly cleared only when the backend process confirms termination.

### 3. Haiku Drafting & Quality Control
- **Multimodal RSS Extraction**: Enhanced `lib/rss-fetcher.ts` with an intelligent parser that scans `content:encoded`, `content`, `summary`, and `description`. Added a robust HTML stripper to provide Clade with clean, high-density news data.
- **Stricter Editor Guardrails**: Updated `lib/llm.ts` with "Strict Source Adherence" guidelines. This prevents Copywriters from hallucinating dates/quotes or using excessive slang that previously caused the Editor to reject 100% of the drafted content.
- **Detailed Rejection Logging**: The Editor now logs the **exact reason for rejection** (e.g., Tone Violation, Hallucination) directly to the system log in the dashboard, providing much-needed transparency into the AI's "thought process."

### 4. Codebase Stabilisation
- **API Response Sync**: Updated `api-client.ts` types to include metadata about the scheduler and current running state.
- **Deduplication Persistence**: Fixed the `articleSourceMap` logic to ensure that even during concurrent (batch) drafting, every article is correctly verified against its specific source material before publication.
